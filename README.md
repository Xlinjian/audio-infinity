# Audio无限+ · Audio Infinity

一款 Chrome (MV3) 扩展，当前稳定版本 **1.0.0**。在**哔哩哔哩**、**百度网盘**、**CCtalk（沪江网校/直播课堂）**在线播放页，以及**本地音频/视频文件**上一键优化音频：**音量均衡、降噪、人声增强、低频/高频优化**；内置**人声清晰度无级调节**、**环绕音效**、**多频段参量均衡器（高级均衡器）**，并支持**分标签页独立音频管理**与**弹窗内联设置**。基于 Web Audio API 实时处理，不改变画面，不影响播放性能。

> 本版本（1.0.0）为**纯音频增强版**：已移除「实时字幕」相关能力（语音识别、Offscreen、字幕浮层、历史与 SRT/TXT 导出），专注把声音处理做到位。需要字幕能力的版本请使用 2.0 分支。

- **✅ 音频能力完整保留**：人声增强、降噪、音量均衡、齿音抑制、空气感、环绕音效、高级均衡器（EQ）、人声清晰度调节、主音量 600% 放大、内置本地播放器、任意网页全局音量增强、分标签页管理、弹窗内联设置——全部保留并可用。
- **🖼 品牌图标统一**：弹窗 / 本地播放器左上角使用扩展正式图标（icons/icon48.png），与商店展示一致。
- **⚙️ 设置内联（不跳转新页面）**：点弹窗右上角「⚙️」即可在弹窗内直接调整主题色、深浅色、逐个开关音频模块；所有改动实时生效并下发到已打开的标签页。

## 核心能力

- **🎚 人声清晰度调节（模糊 ↔ 清亮）**：一条滑块以当前预设参数为「中间档（50）」，向左更暖、更厚、更聚焦人声；向右更亮、更通透、空气感更强。每个预设都以其自身参数作为初始基准，拖动即可在「原始预设」与「更亮/更暖」之间平滑过渡。
- **🔊 环绕音效（MS-Widener）**：宽度滑块将立体声通过 Mid/Side 矩阵拓宽声场。100% 为透明直通（不改变原声），调高则声场更开阔，适合音乐与影视。
- **🗂 分标签页音频管理**：每个浏览器标签页拥有独立的增强状态与设置，互不干扰；工具栏图标徽标按当前标签页实时显示 ON/OFF。
- **⚙️ 设置内联（不跳转新页面）**：点弹窗右上角「⚙️」即可在**弹窗内**直接调整**主题色**（6 款预设 + 取色器）、切换**浅色/深色**主题、逐个**开关音频模块**（人声增强 / 降噪 / 压缩 / 齿音抑制 / 空气感 / 环绕）。设置不再弹出独立网页，所有改动实时生效并下发到已打开的标签页。
- **✨ 极简图形界面**：深色卡片风，自定义滑块填充、SVG 图标，交互清晰高效。
- **🎛 高级参数均衡器（EQ）**：多频段参量 EQ——每频段可自由设置**类型（峰值/低架/高架/低通/高通/带通/陷波）、频率（20Hz~20kHz）、增益（±12dB）、Q 值（0.1~18）**，带**实时频率响应曲线预览**与暖声/明亮/人声突出等一键预设。默认 3 段（低架/峰值/高架），可随时「＋ 添加频段」扩展。弹窗内联显示，内置本地播放器与设置页也内置同一均衡器。参数写入全局设置，所有标签页即时生效。

## 功能

- **一键开关**：点击图标即可开启/关闭当前标签页的音频优化，图标显示 `ON` 徽标。
- **四种预设**：
  - 标准均衡：日常通用，人声与背景兼顾。
  - 人声优先：最大化人声清晰度，适合讲解/教程/播客。
  - 音乐/影视：保留低频与动态，适合音乐与影视区。
  - 强力降噪：抑制环境噪、电流底噪、齿音，适合嘈杂录音。
- **人声清晰度滑块**：以所选预设为基线，无级调节「模糊 ↔ 清亮」（默认 50 = 原预设参数，两端调制幅度已加大，更精细）。
- **环绕音效滑块**：拓宽立体声声场（范围 **100%~300%**，默认 100% = 透明直通）。
- **主音量微调**：站内增强 **50%~600%** 线性增益（Volume Master 风格，最高 6 倍）。
- **高级均衡器（EQ）**：弹窗顶部「设置」左侧的「🎛 高级均衡器」图标按钮，或设置页内嵌的均衡器面板，多频段参量调节，见下。
- **全局音量增强（任意网页）**：在 B站/网盘/本地之外的**任意网页**（如 YouTube、官网视频）也能一键把音量放大到 **最高 600%**。由 popup 通过 `chrome.scripting.executeScript` 注入轻量 booster，自动接管页面内所有 `<video>`/`<audio>`，关闭即恢复原始音量。
- **状态持久化**：设置通过 `chrome.storage.sync` 跨设备同步；每标签页状态独立保存。
- **SPA 适配**：自动捕获 B 站换集/路由切换后重建的视频节点。
- **本地文件支持**：两种方式——① 浏览器直接打开本地文件（`file://`）即时增强；② 内置「本地音频增强器」页面，选择文件即播放增强，最稳、零配置。
- **设置页**：主题色、浅/深色、各模块独立开关（在弹窗内联显示，不跳转新页面）。

## 音频处理链路

```
video → MediaElementSource
      → 参数均衡器 EQ(多频段, 类型/频率/增益/Q, 可关)
      → 高通滤波(降噪, 切低频隆隆)
      → 低频 Shelf(低频优化)
      → 人声 Peaking(3kHz 附近, 人声增强)
      → 高频 Shelf(空气感/清晰度)
      → 齿音 Peaking(-, De-esser)
      → 动态压缩器(音量均衡)
      → Makeup 增益(补偿)
      → 主音量 Gain
      → 环绕 MS-Widener(可选, 拓宽声场)
      → 输出
```

> 参数均衡器（EQ）位于链路最前端，先塑造整体音色，再交给预设的人声/降噪处理，最为自然。
> 清晰度滑块（clarity）会同步调制：人声增益、人声频率、高频空气感、齿音抑制量、低频量与降噪截止频率。
> 环绕（width）仅在「环绕」模块开启且 width > 100% 时生效；width=100% 时为透明直通。

## 安装（开发者模式加载）

1. 打开 Chrome，地址栏输入 `chrome://extensions/`。
2. 右上角打开「开发者模式」。
3. 点击「加载已解压的扩展程序」，选择本 `bilibili-audio-booster-1.0` 文件夹。
4. 打开 B 站视频页、百度网盘播放页、CCtalk 播放页，或直接打开本地文件：
   - B 站：`www.bilibili.com/video/...`
   - 百度网盘：`pan.baidu.com` 中打开任意录音/视频文件预览
   - CCtalk：`*.cctalk.com` 的直播/课程播放页
   - 本地文件：浏览器地址栏直接打开 `file:///.../录音.mp3`（需在扩展详情开启“允许访问文件网址”）
5. 点击工具栏中的扩展图标，打开「音频优化」开关。
6. **首次开启后，点击一次播放器**（浏览器自动播放策略要求用户交互后才能激活音频引擎）。
7. 点弹窗右上角「⚙️」可进入设置页，自定义主题色与模块开关。

## 方式二：内置「本地音频增强器」页面（推荐给本地文件）

最稳、零配置，无需为 `file://` 开权限。从扩展弹窗或扩展管理页进入：

1. 点击扩展图标 → 点「打开内置本地播放器」按钮（或直接访问 `chrome-extension://<id>/player/local-player.html`）。
2. 点击「选择本地文件」，挑一个 mp3/wav/m4a/flac 等音频（也支持带音轨的视频）。也可把文件拖进虚线框。
3. 打开顶部「音频优化」开关，选「人声优先」等预设；用「清晰度」滑块调模糊/清亮，用「环绕」滑块加宽声场，不够响就拉「主音量」。
4. 直接点播放即可听到增强后的声音。

> 该页面在浏览器内实时处理，**不会修改、不会上传**你的原文件，只作用于“正在播放”的声音。

## 方式一：浏览器直接打开本地文件（file://）

1. 扩展管理页（`chrome://extensions/`）→ 找到本扩展 → 点「详情」→ 开启 **“允许访问文件网址”**。
2. 浏览器地址栏输入 `file:///C:/Users/你的用户名/Desktop/录音.mp3` 回车。
3. 点击扩展图标开开关，再点一下播放器激活引擎。

> 注意：部分浏览器对 `file://` 媒体页注入有限制；若无效，请改用上面的「本地音频增强器」页面，效果完全相同。

## 方式三：任意网页音量增强（Volume Master 风格）

适用于 **B站/网盘/本地之外的任意网页**（如 YouTube、官网视频、在线课程），只做音量放大、不做 EQ/降噪：

1. 打开任意网页并播放带声音的视频/音频。
2. 点击扩展图标，弹出窗口里会出现「🌊 全局音量增强」卡片（站内页面则显示「音频优化」卡片）。
3. 打开「音量增强」开关 → 扩展自动把音量放大脚本注入当前标签页（借助 `activeTab` 权限，**无需**额外授权）。
4. 拖动「放大倍数」滑块，最高 **600%**（6 倍）。关闭开关即恢复原始音量。

> 与站内「音频优化」互斥：站内页面已有完整增强（含 600% 主音量），不会出现双注入冲突。

## 设置说明（弹窗内联）

点弹窗右上角「⚙️」可在当前弹窗内展开完整设置页（**不会打开新网页**），页面可上下滑动；再次点击「⚙️」或点「完成」返回主界面：

- **主题色**：6 款预设色 + 自定义取色器，全界面强调色实时生效。
- **深浅色**：深色（默认）/ 浅色一键切换。
- **模块开关**：人声增强、降噪、压缩、齿音抑制、空气感、环绕，可逐个关闭。关闭后对应处理节点变为透明直通，不影响其余效果。
- 所有设置通过 `chrome.storage.sync` 同步，并实时下发到已打开的标签页。
- 如需完整网页版设置，仍可右键扩展图标 →「选项」打开（开发者/高级用途）。

## 高级均衡器（EQ）说明

在弹窗内点「🎛 高级均衡器 (EQ)」按钮，或在设置页内、内置本地播放器内，都会展开**同一套均衡器面板（不打开新页面）**：

- **启用均衡器**：总开关。默认开启且各频段增益为 0（平直、透明），开启后拖动滑块即生效。
- **频段编辑**：每段可单独设置
  - **类型**：峰值 / 低频搁架 / 高频搁架 / 低通 / 高通 / 带通 / 陷波
  - **频率**：20Hz ~ 20kHz（对数滑块，精确选频）
  - **增益**：±12dB
  - **Q 值**：0.1 ~ 18（控制频段宽窄；越大越窄、越尖锐）
  - 每段可「旁路」临时静音、可「✕」删除（至少保留 2 段）
  - 「＋ 添加频段」可增段（默认 3 段，可扩展到任意数量）
- **实时曲线**：面板顶部以 Web Audio `getFrequencyResponse` 绘制**合并频率响应曲线**（含网格与 dB 刻度），所见即所得。
- **快速预设**：平直 / 默认 / 暖声 / 明亮 / 人声突出 / 低音增强，一键套用。
- **全局生效**：改动写入 `chrome.storage.sync`，弹窗、本地播放器、设置页三处面板实时互相同步；在 B 站/网盘/CCtalk/本地/任意网页（全局音量注入也会加载 EQ）均生效。

> 与站内预设、清晰度滑块、环绕可叠加使用：EQ 负责整体音色塑形，预设负责人声/降噪特征，互不冲突。

## 百度网盘使用要点

- 在网盘里**双击**录音文件，会弹出在线播放器（音频常用 `<audio>` 元素，本扩展已支持）。
- 若听不到增强效果，先点一下播放器再开开关；或刷新页面重新加载扩展。
- 网盘在线播放需要文件格式受支持（如 mp3 / m4a / wav / mp4）；不支持的格式请先下载到本地用本地播放器处理。

## 常见问题

- **开了没效果？** 刷新一次页面，再点击播放器任意位置激活引擎；确认弹窗中「引擎」显示为「运行中」。
- **不同标签页设置不一样？** 每标签页独立管理，切换标签页后弹窗显示的是该页状态。
- **声音变小/变大？** 用弹窗里的「主音量」微调，压缩预设会自动补偿。
- **换集/换文件后失效？** 已内置 MutationObserver + 轮询兜底，通常自动恢复；若无，刷新即可。
- **性能影响？** Web Audio 为原生 C++ 实现，CPU 占用极低，不影响播放流畅度。
- **CCtalk 没反应？** 确认在 `*.cctalk.com` 的播放页，且已点开「音频优化」开关并点击播放器激活引擎。CCtalk 播放器常在 iframe 内，本扩展已自动枚举所有帧并聚合媒体计数与增强。
- **媒体数只显示 1？** 弹窗会**跨 iframe 聚合所有播放器的媒体数**（B 站/CCtalk 播放器常位于跨域 iframe），多播放器页面会正确显示总数。
- **EQ 改了没声音变化？** 确认「高级均衡器」里「启用均衡器」已开，且对应频段增益不为 0；改完后所有标签页即时生效，无需刷新。
- **EQ 与预设冲突吗？** 不冲突。EQ 在最前端做整体音色塑形，预设负责人声/降噪，可叠加使用。
- **关闭音频增强后声音发闷？** 关闭时音频走真正旁路（直连扬声器），原声 100% 保真。若仍发闷，请刷新页面后重新加载扩展一次即可。

## 参数调优参考

| 处理项 | 位置 | 作用 |
|---|---|---|
| 高通 80~140Hz | highpass | 去低频隆隆/风噪 |
| 人声 +4~6dB @3kHz | peaking | 凸显人声清晰度 |
| 高频 +2~3dB @8~10kHz | highshelf | 增加空气感 |
| 齿音 -3~5dB @7kHz | peaking(Q高) | 抑制刺耳 s 音 |
| 压缩 3:1~5:1 | compressor | 平衡大小音量 |
| 清晰度滑块 | 综合调制 | 以预设为基线无级调节模糊↔清亮 |
| 环绕 width | MS-Widener | 拓宽声场（>100% 生效） |

可在 `audio-engine.js` 顶部的 `PRESETS` 与 `DEFAULT_MODULES` 中自定义预设参数与默认模块开关。
